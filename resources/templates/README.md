# QA Toolkit Templates\n\nThis directory contains JSON template files that define the structure and fields for various quality assurance documents and forms used in the VelocityQA toolkit.\n\n## Template Structure\n\nEach template follows this basic structure:\n\n```json\n{\n  \"templateName\": \"Template Name\",\n  \"version\": \"1.0\",\n  \"fields\": {\n    \"fieldId\": {\n      \"label\": \"Field Label\",\n      \"type\": \"text|textarea|select|date|number\",\n      \"placeholder\": \"Placeholder text\",\n      \"required\": true|false,\n      \"options\": [\"Option1\", \"Option2\"] // Only for select type\n    }\n  },\n  \"structure\": {\n    \"sections\": [\n      {\n        \"title\": \"Section Title\",\n        \"fields\": [\"fieldId1\", \"fieldId2\"]\n      }\n    ]\n  }\n}\n```\n\n## Available Templates\n\n1. **Bug Report Template** (`bug-report-template.json`)\n   - A structured template for documenting bugs that makes reproduction easy and speeds up the debugging process.\n\n2. **Test Coverage Report** (`coverage-report-template.json`)\n   - A one-page summary template designed for stakeholders to understand test coverage and quality status at a glance.\n\n## Adding New Templates\n\nTo add a new template:\n\n1. Create a new JSON file following the structure above\n2. Name it using kebab-case (e.g., `template-name.json`)\n3. The template will be automatically loaded by the application\n\n## Template Features\n\n- User can fill in custom values in the template\n- Templates can be downloaded in various formats (DOCX, PDF, Markdown, TXT)\n- Template structure is preserved in all download formats
